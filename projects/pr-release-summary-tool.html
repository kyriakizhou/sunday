PR Release Summary Tool

release engineering / ai summarization / risk analysis / java / slack automation

Because of my organization's release cadence, there can be at least a three-week gap between an engineer's code commit and its deployment to production. I built an AI release summary and risk analysis tool to act as a final checkpoint before each release.

The tool reads pull-request diffs for the release, generates concise summaries of important code changes, highlights risky or operationally sensitive changes, and recommends Splunk and Hubble monitoring for release day.

Architecture

  • Built asynchronous processing in Java with CompletableFuture so release summaries could be generated across many teams and engineers without blocking on one slow model call.
  • Integrated with the existing release schedule in the codebase, where the application bundle scheduler runs every day.
  • Cached successfully generated PR summaries so reruns did not waste model calls or rewrite stable release output.
  • Sent automatic Slack notifications to me when generation failed, turning errors into visible follow-up instead of silent release-report gaps.

Context Window Design

The hardest design constraint was fitting real production diffs into model context windows. I first tried sending all engineers' PR diffs for a team in one prompt. That was simple, but it did not scale. I then moved to one prompt per engineer, and finally to multi-batch truncation for engineers with larger diff sets.

That chunking design made the system future-ready: the tool could handle larger teams, noisier release trains, and unusually active engineers while still preserving enough code context for useful summaries.

Generated Report

  • Important code changes: changes are grouped by feature. Each bullet includes a PR link, the engineer's email, and a one-sentence AI-generated summary.
  • Risk analysis: the report highlights the most critical changes, including feature launches, global property enablement, and backward-incompatible changes.
  • Release-day monitoring: the report suggests Splunk and Hubble dashboards, alerts, and signals to watch during rollout.

Adoption

The tool has been running in production to generate release summaries for 33 teams and 300+ engineers every release. I presented it to the org VP and recommended adoption by parallel ASE Commerce organizations.

I also expanded it for QA and testing engineers, allowing them to sign up for summaries of code changes from the engineers they work with most closely.
