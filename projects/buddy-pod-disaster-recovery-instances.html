Buddy Pods

infrastructure / disaster recovery / automatic app health check / instance failover

Buddy Pods was a 2026 infrastructure project for using 350 idle disaster recovery standby instances during normal operation without compromising their real failover role. For the application involved, DR instances in one pod could process work for a paired active pod in the same datacenter.

If the primary datacenter went down, those standby instances would automatically shut down and restart in their real DR role. That preserved failover readiness while letting otherwise idle capacity help with live backlog, latency, and traffic spikes.

The pairing analysis was bidirectional, balanced by instance count, and matched by data hall. After rollout, the system cleared millions of queued jobs within hours while keeping the hardware capacity required for a full failover intact.
